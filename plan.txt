# Plan: Mock → Firebase Migration + Discover Gender Filter + Register DatePicker

## Context

Eşleşbulus (React Native + Expo + Firebase) dating app has 14+ files importing from `constants/mockUsers.ts`. Firebase (Auth + Firestore + Storage) is already set up in `config/firebase.ts`. `AuthContext` already subscribes to `users/{uid}` Firestore doc — partial real-user infra exists, but discovery / lists / single-user fetches all read from MOCK_USERS arrays. This plan does three things:

1. **Migrates all mock reads to Firestore** (hooks for list + single user), migrates Interactions & BlockedUsers contexts to subcollections, deletes `constants/mockUsers.ts` entirely.
2. **Adds opposite-gender filter** in Discover — current user's gender determines who they see by default.
3. **Adds DatePicker to register** with age computation + 18+ enforcement. Removes the manual GG/AA/YYYY birth-date inputs from `profile-setup.tsx` (single source of truth on registration).

Why now: `MockUser` is the #2 god node with 14 cross-community edges (mock data leaks into production screens). Removing it lets the app run with real Firestore users and removes a major architectural smell flagged by graphify.

---

## Decisions (locked by user)

- **MOCK_USERS:** delete entirely, no seed, no fallback. Discover will be empty until real users register.
- **Birthdate:** register-only. profile-setup loses its manual date inputs.
- **Interactions + BlockedUsers:** migrate to Firestore now (subcollections).

Extend `UserProfile` type to include all discovery-needed fields:
```ts
export type UserProfile = {
  uid: string;
  name: string;
  email: string;
  photoURL?: string;
  birthDate?: string;   // ISO yyyy-mm-dd (from DatePicker)
  age?: number;         // computed at register, stored

  bio?: string;
  city?: string;
  photos?: string[];
  interests?: string[];
  job?: string;
  height?: number;
  online?: boolean;
  lastActive?: number;  // unix ms; serverTimestamp on writes
  verified?: boolean;
  vip?: boolean;
  profileComplete: boolean;
};
```

### `hooks/useUsers.ts` (NEW)
Real-time subscription to `users` collection with client-side filtering.
- Returns `{ users, loading, error }`.
- Filters out current user's own doc.
- Caller passes `Filters` from `constants/filters.ts`; hook applies same predicate as `applyFilters()`.
- Uses `onSnapshot(collection(db, "users"))` so new registrations appear live.

### `hooks/useUser.ts` (NEW)
Single-user `onSnapshot(doc(db, "users", uid))` subscription. Replaces every `getUserById(id)` call. Returns `{ user, loading }`.

### Firestore schema
```
users/{uid}                       // existing, extended
users/{uid}/sent_his/{otherUid}   // NEW — { text, emoji, at }
users/{uid}/liked/{otherUid}      // NEW — { at }
users/{uid}/blocked/{otherUid}    // NEW — { name, photo, at }
```

### `lib/age.ts` (NEW)
Small util:
```ts
export function calculateAge(birthDate: Date): number {
  const today = new Date();
  let age = today.getFullYear() - birthDate.getFullYear();
  const m = today.getMonth() - birthDate.getMonth();
  if (m < 0 || (m === 0 && today.getDate() < birthDate.getDate())) age--;
  return age;
}
```

Replace `MOCK_USERS` / `MockUser` / `getUserById` everywhere:

### List consumers — swap to `useUsers()` (or specialized variants)
- `app/(tabs)/index.tsx` → `const { users } = useUsers(filters)` (filters applied inside hook).
- `app/(tabs)/chat.tsx` → for now show recent chats from a future `messages` collection; until then show `useUsers().slice(0, 6)` as a "Mesaj başlat" suggestion list.
- `app/(tabs)/matches.tsx` → same temporary approach until matches collection exists; show liked-me derived from `users/{uid}/liked` reverse-lookup later.
- `components/discover/StoriesBar.tsx` → drop entirely OR fetch users with `hasStory: true`. Recommendation: drop stories UI from discover until stories backend exists.
- `components/discover/VipSection.tsx` → `useUsers({ ...DEFAULT_FILTERS, gender: oppositeGender }).filter(u => u.vip)`.
- `constants/notifications.ts` → currently exports `MOCK_NOTIFS` referencing mock IDs. **Delete this constant** — notifications need their own backing later. Stub the notifications popup to show "Henüz bildirim yok".

### Single-user consumers — swap to `useUser(uid)`
- `app/user/[id].tsx`, `app/chat/[id].tsx`, `app/call/[id].tsx`, `app/story/[id].tsx` — replace `getUserById(id)` with `useUser(id)`.
- Components receiving a `MockUser` prop (`ProfileCardAlbum`, `ProfileCardList`, `HiMessageModal`) — change prop type from `MockUser` to `UserProfile`. Field surface is compatible (same names).

### `context/InteractionsContext.tsx` — Firestore-backed
Replace local state with Firestore subscriptions:
- `sentHis`: `onSnapshot(collection(db, "users", uid, "sent_his"))`.
- `likedUsers`: `onSnapshot(collection(db, "users", uid, "liked"))`.
- `sendRandomHi(otherUid)`: `setDoc(doc(db, "users", uid, "sent_his", otherUid), { text, emoji, at: serverTimestamp() })`.
- `likeUser(otherUid)`: same shape, `liked` collection.
- Daily-limit count: `query(..., where("at", ">=", startOfToday))`.

### `context/BlockedUsersContext.tsx` — Firestore-backed
- `blocked`: `onSnapshot(collection(db, "users", uid, "blocked"))`.
- `blockUser(user)`: `setDoc(..., { name, photo, at: serverTimestamp() })`.
- `unblockUser(uid)`: `deleteDoc(...)`.

### DELETE
- `constants/mockUsers.ts` (entire file — Gender type re-exported from filters.ts)
- `constants/notifications.ts` `MOCK_NOTIFS` constant (keep file empty or delete if unused)
- All `import ... from "@/constants/mockUsers"` lines.

## Phase 5 — Firestore security rules (CRITICAL)

`firestore.rules` (create if missing — currently no rules file in repo):
```
match /users/{uid} {
  allow read: if request.auth != null;
  allow write: if request.auth.uid == uid;
  match /{sub=**}/{doc} {
    allow read, write: if request.auth.uid == uid;
  }
}
```
Push with `firebase deploy --only firestore:rules`. **Without this, anyone can read/write any user.**

## Verification

1. **Build & run:** `npx expo start --clear` — TypeScript compiles, no stale `MockUser` imports.
2. **Token check:** `rg "MOCK_USERS|MockUser|getUserById|mockUsers" --type tsx --type ts` should return only the deleted-file path or zero hits.
3. **Register flow:** Create new account → pick birthdate via picker → confirm age shown → submit. Verify `users/{uid}` doc in Firestore Console has `birthDate`, `age`, `profileComplete: false`.
4. **Profile setup:** Continue flow → gender + city + interests → no birthdate step. Verify `profileComplete: true`.
5. **Discover gender filter:** Log in as a "Kadın" account → Discover shows only "Erkek" users (and vice versa). Open FilterSheet → switch to "Tümü" → mix appears.
6. **Single-user fetch:** Tap a profile card → `app/user/[id]` loads via `useUser(id)` Firestore subscription. Edit that user's bio in another browser tab → live update in app.
7. **Interactions:** Send a "Hi" → check Firestore Console for `users/{senderUid}/sent_his/{receiverUid}` doc with text/emoji/at timestamp.
8. **Blocked users:** Block someone → check `users/{uid}/blocked/{otherUid}` exists. Reopen app → list still shows the block (persistence proven).
9. **Security rules:** In Firestore Console rules playground, attempt unauthenticated read of `users/x` → denied. Authenticated read of `users/x` → allowed. Authenticated write to `users/y` where `y != uid` → denied.
10. **Empty-state UX:** If Firestore `users` collection has only your own doc, Discover should show an empty state ("Şu an kimse yok") — verify a graceful empty render exists in `app/(tabs)/index.tsx`.
